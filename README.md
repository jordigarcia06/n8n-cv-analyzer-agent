# 🤖 AI CV Analyzer Agent

An agentic automation that processes CVs in PDF format, extracts structured data using LLaMA 3.3 70B, scores candidates and saves results to Airtable — all in under 10 seconds.

Built with n8n (self-hosted), Groq API and Airtable.

## How it works

1. Recruiter uploads a CV (PDF) via a web form generated by n8n
2. The system extracts text from the PDF automatically
3. An AI Agent (LLaMA 3.3 70B via Groq) analyzes the profile and returns structured JSON
4. A code node parses and sanitizes the output
5. Results are saved to Airtable as a new record

```
Form Trigger → Extract from PDF → AI Agent → Code (parse) → Airtable
```

---

## Output fields

| Field | Description |
|-------|-------------|
| `nombre` | Candidate full name |
| `email` | Email address — `NULL` if not found in the CV |
| `puntuacion` | Score from 1 to 10 based on seniority and skills |
| `skills` | Technical skills extracted, comma-separated |
| `resumen` | 2–3 sentence professional summary |

- **[Airtable](https://airtable.com)** — structured data storage

---

## Key technical decisions

**Prompt engineering to prevent hallucinations**
The system prompt includes explicit negative instructions to prevent the model from inventing data not present in the CV — particularly for the email field. If no email is found, the agent returns `null`.

**Markdown stripping in code node**
LLMs sometimes wrap JSON output in markdown code blocks. A sanitization step removes backticks before parsing to prevent JSON.parse failures.

**Graceful handling of missing fields**
The code node applies default values for null fields before sending data to Airtable, preventing record creation errors.

## Setup

### Requirements

- n8n instance (self-hosted or cloud)
- [Groq API key](https://console.groq.com) — free tier available
- [Airtable Personal Access Token](https://airtable.com/create/tokens)

### Airtable token scopes needed

- `data.records:read`
- `data.records:write`
- `schema.bases:read`

### Airtable table structure

| Field | Type |
|-------|------|
| nombre | Single line text |
| email | Single line text |
| puntuacion | Number |
| skills | Long text |
| resumen | Long text |

### Import the workflow

1. Download `workflow.json`
2. In n8n go to **Workflows → Import from file**
3. Add your credentials: Groq API key + Airtable token
4. Activate the workflow

---

## Known limitations

- PDFs generated via browser print ("Print to PDF") may not have embedded text and will return empty extraction
- Scanned PDFs (image-based) are not supported — OCR integration planned as next iteration
- Groq free tier has rate limits — suitable for low-volume use cases
